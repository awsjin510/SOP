# 使用者流程（UX Flows）

定義所有頁面與互動流程。

## 主要流程

```
┌─────────────┐
│  Landing    │（未登入）
│  /          │
└─────────────┘
       ↓ 登入
┌─────────────┐
│  Dashboard  │ ← 主頁，所有 SOP 列表
│  /dashboard │
└─────────────┘
   ↓     ↓     ↓
   ↓     ↓     ↓
[+ 新增]  [檢視]  [更新]
   ↓     ↓     ↓
┌──────┐ ┌──────┐ ┌──────┐
```

## 頁面清單

| Route | 頁面名稱 | 用途 |
|-------|---------|------|
| `/` | Landing | 未登入時的歡迎頁 |
| `/login` | Login | 登入頁 |
| `/dashboard` | Dashboard | 主頁（SOP 列表） |
| `/sop/new` | NewSop | 新建 SOP（上傳素材） |
| `/sop/:id` | SopDetail | SOP 詳細頁（版本、預覽） |
| `/sop/:id/update` | UpdateSop | 更新 SOP（上傳更新素材） |
| `/sop/:id/versions/:versionId` | VersionDetail | 特定版本檢視 |
| `/sop/:id/versions/:versionId/diff/:otherVersionId` | VersionDiff | 版本比對 |
| `/job/:jobId` | JobProgress | 處理進度頁 |
| `/job/:jobId/review` | ReviewInterface | 審核介面 |
| `/settings` | Settings | 個人設定（用量、偏好） |

## 詳細流程

### Flow 1：第一次使用

```
1. 使用者開啟 https://your-app.web.app
2. 看到 Landing 頁
   → 「歡迎使用 SOP System」
   → 「Google 登入」按鈕
3. 點擊登入 → Firebase Auth 彈出 Google 登入
4. 登入成功 → 重導到 /dashboard
5. Dashboard 是空的 →「您還沒有 SOP，[+ 建立第一份]」
```

### Flow 2：建立第一份 SOP

```
1. /dashboard 點 [+ 新增 SOP]
2. 進入 /sop/new
   → 看到上傳區（拖拉或點擊選檔）
   → 看到「步驟 1/4：上傳素材」進度標示

3. 上傳檔案：
   → 拖拉訪談.txt → 系統自動分類為 "transcript"
   → 拖拉操作截圖.png → 系統分類為 "screenshot"
   → 拖拉 manual.docx → 系統分類為 "document"
   → 即時顯示已上傳清單
   → 顯示總大小、檔案數

4. 確認分類（步驟 2/4）：
   → 系統依檔名/內容判斷的類型，使用者可調整
   → 範例：
     - interview-aws.txt → 訪談逐字稿 ✓
     - manual.docx → 既有文件 ✓
     - screenshot-001.png → 截圖 ✓

5. 設定基本資訊（步驟 3/4）：
   → 標題：（自動建議或手動輸入）
   → 適用對象：（從偏好設定預填）
   → 預估學習時間：（可空白讓系統估）
   → 分類：選 dropdown
   → 難度：初級/中級/進階
   → 進階選項（可摺疊）

6. 確認與啟動（步驟 4/4）：
   → 預覽所有設定
   → 預估處理時間：~8 分鐘
   → 預估 API 成本：~$3 USD
   → [開始處理] 按鈕

7. 點擊後：
   → 上傳檔案到 Firebase Storage
   → 建立 processing_job 文件
   → 重導到 /job/{jobId}
```

### Flow 7：更新 SOP

```
1. /sop/{sopId} 點 [更新]
2. 進入 /sop/{sopId}/update
3. 看到「您正在更新：EC2 啟動 SOP v1.3.0」
4. 上傳更新素材（流程同 Flow 2 上傳階段）：
   → 修改清單.docx
   → 會議紀錄.txt
   → 新截圖
5. 設定變更摘要（一句話描述本次更新）
6. [開始更新] 啟動處理
7. 進入 /job/{jobId} 看進度
8. 同 Flow 4 進行審核
9. 完成後重導到 /sop/{sopId}（顯示新版本）
```

## 互動細節

### 拖拉上傳的處理

```
1. 拖拉到頁面任何位置 → 顯示全螢幕的拖拉提示
2. 鬆開後 → 自動分類（依副檔名與內容）
3. 顯示分類結果 → 使用者可調整
4. 上傳開始 → 顯示進度（每個檔案獨立進度條）
```

### 進度同步機制

所有處理進度透過 Firestore listener 即時同步：
- UI 不用 polling
- 多分頁開啟同一任務都能即時看到進度
- 可在不同裝置追蹤進度

### 錯誤處理

每種錯誤類型都有對應 UI：
- **網路錯誤**：橘色 banner，提供「重試」按鈕
- **檔案錯誤**：行內紅字，說明哪個檔案有問題
- **API 錯誤**：彈窗，含錯誤碼與重試選項
- **配額超限**：紅色 banner，引導到設定頁

## 響應式設計

主要支援桌面（1280px+）。

行動裝置：
- 主要功能可用，但複雜操作（審核介面、版本比較）建議用桌面
- 至少要能看 SOP 列表、下載檔案、看通知
